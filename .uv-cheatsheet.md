# uv Quick Reference for cb-text-analytics

This project supports **uv** for fast, modern Python package management.

## ðŸ“¦ Common Commands

### Running Python Scripts
```bash
# Run the quick analysis script
uv run python src/quick_analysis.py

# Run any Python command
uv run python -c "import pandas; print(pandas.__version__)"
```

### Running Jupyter
```bash
# Start Jupyter notebook
uv run jupyter notebook

# Or Jupyter lab
uv run jupyter lab
```

### Managing Dependencies

```bash
# Install new package
uv add package-name

# Install dev dependency
uv add --dev pytest

# Update all dependencies
uv sync

# Update specific package
uv add package-name@latest
```

### Virtual Environment

```bash
# uv automatically manages the venv in .venv/
# To activate manually (usually not needed):
source .venv/bin/activate  # Linux/Mac
.venv\Scripts\activate     # Windows
```

### Python Version

```bash
# Use the Python version from .python-version (3.11)
uv run python --version

# Install specific Python version
uv python install 3.11
```

## ðŸ’¡ Why uv?

- **10-100x faster** than pip
- **Automatic venv** management (no `venv` or `activate` needed)
- **Better dependency resolution**
- **Project isolation** by default
- **Modern standard** (pyproject.toml)

## ðŸ”„ Switching from pip

Already using pip? No problem!

```bash
# Old way:
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python src/quick_analysis.py

# New way:
uv sync
uv run python src/quick_analysis.py
```

Both work! We maintain both `requirements.txt` and `pyproject.toml`.



```bash

```
